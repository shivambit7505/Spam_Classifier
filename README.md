# SMS Spam Detector — Flask API + independent frontend

QSkill AI/ML Internship — Task 2.

A real TF-IDF + Logistic Regression classifier trained on the genuine UCI
SMS Spam Collection dataset (5,572 real text messages: 4,825 ham, 747 spam),
served through a Flask REST API, consumed by a standalone HTML/CSS/JS
frontend — no build step, no framework.

## Architecture

```
spam-app/
├── backend/
│   ├── train_model.py      # downloads dataset, trains + persists the model
│   ├── app.py                # Flask API
│   ├── requirements.txt
│   ├── model.joblib            # generated by train_model.py
│   ├── metadata.json            # generated
│   └── sample_messages.json     # generated
└── frontend/
    └── index.html             # standalone — open directly or serve statically
```

## Run it

### 1. Backend

```bash
cd backend
python -m venv venv && source venv/bin/activate   # optional
pip install -r requirements.txt
python train_model.py     # downloads the dataset (first run only), trains, saves model.joblib
python app.py               # starts the API on http://127.0.0.1:5001
```

```bash
curl http://127.0.0.1:5001/api/health
# {"status": "ok", "model": "logreg", "test_accuracy": 0.9704...}
```

### 2. Frontend

In a separate terminal:

```bash
cd frontend
python -m http.server 8081
```

Open `http://127.0.0.1:8081`. Type or paste a message, or click a real
example chip pulled from the test set, and classify it.

## API reference

| Method | Path             | Description                                                  |
|--------|------------------|---------------------------------------------------------------|
| GET    | `/api/health`    | Service status, active model, test accuracy                    |
| GET    | `/api/metadata`  | Accuracy/precision/recall/F1, confusion matrix, top spam/ham words |
| GET    | `/api/samples`   | Real test-set messages with true labels (for try-it chips)      |
| POST   | `/api/predict`   | Body: `{"message": "..."}` → prediction, confidence, per-word contributions |

```bash
curl -X POST http://127.0.0.1:5001/api/predict \
  -H "Content-Type: application/json" \
  -d '{"message": "URGENT! You have won a free prize, claim now, reply STOP to opt out"}'
```

## Model details

- Dataset: 5,572 real SMS messages (UCI SMS Spam Collection), 86.6% ham / 13.4% spam
- Split: 80/20 stratified, `random_state=42`
- Pipeline: `TfidfVectorizer(stop_words="english", max_features=5000)` → `LogisticRegression`
- Test results: **97.0% accuracy, 100% precision, 77.9% recall, 87.5% F1**
  - 100% precision means: in testing, it never flagged a real message as spam (zero false positives)
  - 77.9% recall means: it catches about 4 out of every 5 actual spam messages
  - A `LinearSVC` scored slightly higher overall (98.2% accuracy) but lacks
    `predict_proba`, so Logistic Regression ships as production to support
    confidence scores in the UI — full comparison is in `train_model.py`'s
    output and `metadata.json`.
- Per-word explainability is real model introspection: each flagged word's
  "weight" is its actual learned logistic-regression coefficient (positive
  pushes toward spam, negative toward ham), not a heuristic guess.

## Notes

- `/api/predict` validates input server-side (missing/non-string/empty/too-long
  message) and returns `400` with details on failure.
- The training data has some HTML-entity artifacts (e.g. `&lt;#&gt;` for
  redacted numbers) inherited from the original dataset; the `/api/samples`
  endpoint unescapes them before returning.
